VL6180X by Pololu is a great sensor module if you need accurate distance measurements for your robots. Specially, if you requirement is within 10mm - 200mm. However, interfacing multiple VL6180X sensors with Arduino is not straightforward.

In these VL6180x modules, the communication protocol is I2C. The advantage of having such a communication protocol is, we can use same communication bus to communicate with multiple modules. On the other hand, to successfully communicate with multiple modules, first we need to identify those modules individually. To do that, we use unique addresses for each of the modules. After that, by referring to that unique address we can communicate with multiple modules without any interruption.

First, let's see how pins are connected with Arduino

VL6180X > Arduino
VIN > 5v
GND > GND
SDA > SDA
SCL > SCL
GPIO0 > Any digital/analog pin

SDA and SCL are common for all the sensors. But GPIO0 should be connected individually to the Arduino board. Here, we will be using 5 VL6180X sensors with a Arduino Uno.

Now, the problem is how to unique addresses for these 5 sensors. VL6180X sensors by Pololu have this pin called GPIO0 and here we will be using this pin to make the sensors hardware stand-by(usually other sensor manufacturers name this pin as XSHUT). By default GPIO0 is at VDD voltage level(HIGH). By making this pin LOW we can put the sensor to stand-by mode.

To write addresses to all the five sensors, first by making all the 5 GPIO0 pins LOW, we put sensors into stand-by mode. After that, by making each pin HIGH one by one we can write I2C addresses to the sensors(here the addresses are 43,44,45,46,47).

Sharp IR of ToF ? It took us almost one week to come to a decision and finally we decided to go with ToF's. Wanna know why?

When selecting sensors for a micro mouse there are certain features that we should consider.

  1. Range of measurements
  2. Accuracy
  3. Speed
  4. Size of the sensor

However. both sharp IR and ToF sensors have above features and that makes difficult to take a decision.

For the micro mouse that we were building, we wanted the sensors to be able take measurements in the range 10mm - 200mm with very high accuracy. Since we were not planning to run the micro mouse in very high speeds, the speed of the sensors was not a concern. But we wanted our micro mouse to be small as possible and since were planning to use five sensors, size of the sensors was one of the major concerns.

For all our requirements, we found that VL6180X by Pololu is the one we needed. However, the only issue we had with sharp IR sensors is their size. (we were looking at the sharp IR sensor modules. Physical size won't matter if IR emitters and receivers are taken separately)

In one of the next posts, I'll be talking about interfacing multiple VL6180X with STM32F103C8 using Arduino. Stay tuned for that.

Team Piccola ! Four friend building a micro mouse called "piccola" for a interuniversity competition, ROBOFEST. Wanna build a Piccola ?

When we first started to build a micro mouse, the main concern was to find a suitable micro-controller. There were few features that we looked for when selecting a micro-controller.

  1. Physical size
  2. Memory
  3. Number of GPIO pins
  4. Speed
  5. Easy to use

To be honest, we couldn't find a micro-controller which ticks all the boxes for us. But there were few micro-controller that we Â found very useful in this case.

  1. STM32F103C8 - Blue pill
  2. Arduino pro mini
  3. Arduino Geniuno Uno
  4. Arduino nano

In the Arduino boards that we found, there were few disadvantages when used in a micro mouse. Some were large in size(Arduino Uno), some didn't have enough GPIO pins(specially external interrupt pins ) and some didn't have enough memory to store the micro mouse maze solving algorithm.

After two three rounds of discussions in the team, we decided to use STM32 blue pill as the micro-controller. The initial concern was how to program this blue pill. Luckily, STM32 is programable trough the very familiar Arduino IDE. Overall STM32 is a good choice when compared with others. The small physical size and more GPIO pins are very valuable when building a micro mouse.

Since this is Arduino compatible, we can use all the libraries available for Arduino without any problem(STM32 programing will be talked in a separate post ). Â
