Team Piccola ! Four friend building a micro mouse called "piccola" for a interuniversity competition, ROBOFEST. Wanna build Piccola ?

When we first started to build a micro mouse, the main concern was to find a suitable micro-controller. There were few features that we looked for when selecting a micro-controller.

  1. Physical size
  2. Memory
  3. Number of GPIO pins
  4. Speed
  5. Easy to use

To be honest, we couldn't find a micro-controller which ticks all the boxes for us. But there were few micro-controller that we  found very useful in this case.

  1. STM32F103C8 - Blue pill
  2. Arduino pro mini
  3. Arduino Geniuno Uno
  4. Arduino nano

In the Arduino boards that we found, there were few disadvantages when used in a micro mouse. Some were large in size(Arduino Uno), some didn't have enough GPIO pins(specially external interrupt pins ) and some didn't have enough memory to store the micro mouse maze solving algorithm.

After two three rounds of discussions in the team, we decided to use STM32 blue pill as the micro-controller. The initial concern was how to program this blue pill. Luckily, STM32 is programable trough the very familiar Arduino IDE. Overall STM32 is a good choice when compared with others. The small physical size and more GPIO pins are very valuable when building a micro mouse.

Since this is Arduino compatible, we can use all the libraries available for Arduino without any problem(STM32 programing will be talked in a separate post ).  

This is a series of posts where I'm trying to talk about building a micro mouse from scratch. If  there's something in this post unclear to you, please wait for the next set of posts or hit me up on the comments.
